FLOW SUMMARY

Phase 1 — Builder voice: User opens /builder → BuilderChat mounts → useBuilderChatVoice auto-starts Vapi SDK with the builder assistant ID → user speaks → transcript streams into chat bubbles in real-time. Text typed in the input box is shown as bubbles locally but makes no backend calls; it becomes a fallback transcript.
Phase 2 — Transcript commit: When the Vapi call ends, POST /builder/transcript/commit is fired with the call_id and/or client transcript. The server optionally fetches the official transcript from Vapi's REST API, normalises duplicates, and translates to English if needed.
Phase 3 — Extraction: run_draft_single() runs: (1) rewrite via LLM, (2) extract with PROMPT_EXTRACT_SINGLE_CARDS, (3) parse JSON into a Family, (4) inject UUIDs + metadata, (5) persist to DB with visibility=False.
Phase 4 — Finalize: _finalize_card() flips experience_card_visibility=True then embed_experience_cards() calls the embedding provider to generate a 324-dim vector stored in pgvector. The card is now searchable.
Phase 5 — Cards page: Frontend invalidates React Query caches → GET /me/experience-card-families fetches the new card. User sees SavedCardFamily cards. Inline edits go to PATCH /experience-cards/{id}. Clicking the pencil icon navigates to /cards/{id}/enhance and pre-warms the Vapi SDK.
Phase 6 — Enhance: The enhance page loads the card into local draft state. Voice uses a separate edit assistant (NEXT_PUBLIC_VAPI_EDIT_ASSISTANT_ID) with the card snapshot injected as variableValues. Structured tool payloads from Vapi are parsed client-side into patches applied to local state — no DB write until Save. Typed chat calls POST /experience-cards/fill-missing-from-text and merges the response locally. "Save & exit" calls POST /experience-cards/{cardId}/commit-draft which applies the parent patch, upserts all children (update by id or upsert by type), re-embeds parent + all children, invalidates locale cache, and returns a CardFamilyResponse.
